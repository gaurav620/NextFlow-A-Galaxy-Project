# Galaxy.ai Developer Standards (Mandatory)

## Our Philosophy

- **Lean & Efficient Operations** — No unnecessary bureaucracy or micromanagement.
- **Innovation Over Process** — Creativity, problem-solving, autonomy > rigid procedures.
- **No Hand-Holding** — Small team, everyone independent. Take initiative, own your work.

## Technical Standards (Mandatory — submissions that don't meet these are rejected)

### 1. Preparation
- Study the **Dub GitHub Repository** (https://github.com/dubinc/dub) as the reference for coding practices, structure, and documentation.

### 2. Coding Standards
- **Next.js Best Practices** — https://nextjs.org/docs/pages/building-your-application/deploying/production-checklist
- **Readable Code** — Clean, modular, consistent naming, proper indentation.
- **Document Your Work** — Clear comments/inline docs for complex logic.

### 3. TypeScript
- Always use strict type-checking.

### 4. Styling
- Use **TailwindCSS** for all CSS.
- Use **ShadCN** for pre-designed components.

### 5. Testing
- Test thoroughly before submission (functionality, edge cases, responsiveness).

## Code Review & Submission

- **High Standards Are Mandatory** — deviations get rejected outright.
- **Self-Review First** — test & review before submitting.
- **Be ready to defend decisions** and incorporate feedback in peer reviews.
